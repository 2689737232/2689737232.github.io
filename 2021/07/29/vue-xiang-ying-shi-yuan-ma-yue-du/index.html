拉取源代码

从官方仓库拉取了vue的源码，自己做了一点理解笔记。

1. 响应式

具有响应式的对象，或他的某个属性，发生变化后能够自动运行一些函数。
通过Object.definedProerty()函数的get方法中使用Dep实例中的depend方法收集依赖，
通过set方法触发实例中的notify。notify会重新执行更新函数。

2. 官网对响应式原理的解释

  • 把实例data中的所有属性，通过Object.defineProperty重写这些属性的getter和setter。通过在get中收集依赖，通过set去触notify方法。notify方法会通知watcher。
  • 每个组件实例都对应一个 watcher 实例,被notify后会重新触发render函数。

2. 相关类、方法

  • Observer
  • Dep
  • watcher
  • scheduler

2.1 Observer

data传入new Vue({data:xxx})初始化，是在callHook(vm, 'beforeCreate')之后callHook(vm, 'created')之前。

initLifecycle(vm) // 在vm上添加了一些属性，$parent、$root、$refs、_isMounted、_isDestroyed 
initEvents(vm)   
initRender(vm)   // 初始化渲染函数，将$createElement添加到vm上。对$attrs、$listeners在开发和生成环境做不同的响应式处理
               // $createElement生成 VNode，虚拟 DOM相当于VNode树
callHook(vm, 'beforeCreate')  // 调用声明周期函数beforeCreate

initInjections(vm) // resolve injections before data/props  在数据建立前，初始化解析注入
initState(vm)      // 初始化数据通过观察者模式建立，在这里对实例的data使用到了observe
                  // 初始化实例配置中的 data、props、methods、computed
                  // 在对data进行初始化时判断是否已经定义了名字相同的props和methods

initProvide(vm) // resolve provide after data/props  在数据初始化后，解析provide 

callHook(vm, 'created')  // 在数据完成初始话后，数据可以访问了但是，真实dom还未被替换

在observer包下的index.js中。

在index.js中有Observer类和observe、defineReactive方法等。

通过observe函数可以将一个对象中的每个属性，添加get和set。observe函数返回一个Observer实例。Vue.observable中也使用到了observe函数。

new Vue({..., data:{}})中data会被传入observe函数。observe函数只会对对象、对象的属性和数组添加get和set。数组中的元素没有添加，所以对数组中的元素进行操作需要使用数组方法。

observe函数:

export function observe (value: any, asRootData: ?boolean): Observer | void {
  if (!isObject(value) || value instanceof VNode) {
    return
  }
  let ob: Observer | void
  if (hasOwn(value, '__ob__') && value.__ob__ instanceof Observer) {
    ob = value.__ob__
  } else if ( 
    shouldObserve &&
    !isServerRendering() &&
    // 过滤掉不是数组和对象的值。
    (Array.isArray(value) || isPlainObject(value)) &&
    Object.isExtensible(value) &&
    !value._isVue
  ) {
    ob = new Observer(value) 
  }
  if (asRootData && ob) {
    ob.vmCount++
  }
  return ob
}

Observer类：

export class Observer {
  value: any;
  dep: Dep;
  vmCount: number; // number of vms that have this object as root $data
  constructor (value: any) {
    this.value = value
    this.dep = new Dep()
    this.vmCount = 0
    def(value, '__ob__', this)  // 通过Object.definedProperty 定义对象value身上的__ob__
                                // 将响应式对象上的__ob__指向当前实例

    if (Array.isArray(value)) {
      // 如果当前环境中可以通过 {}.__proto__ 取到，将Array.prototype上的方法
      // 通过Object.create的方式赋值给对象上的__proto__
      // 如new Vuew({data})的data中有个arr的属性，是一个数组那么这个数组的__proto__会有
      //  pop push reverse shift sort splice unshift方法       
      // arrayMethods在Vue中定义为：       
      //    const arrayProto = Array.prototype
      //    const arrayMethods = Object.create(arrayProto)
      if (hasProto) { 
        protoAugment(value, arrayMethods)
      } else {
         // 如果环境不支持通过[].__proto__获取原型，Vue直接在数组属性上添加
        copyAugment(value, arrayMethods, arrayKeys)
      }
      // 然后在对数组中的每一项进行递归式的生成get set
      this.observeArray(value)
    } else {
      // 否则如果是其他的值
      this.walk(value)
    }
  }
  walk (obj: Object) {
    const keys = Object.keys(obj)
    for (let i = 0; i < keys.length; i++) {
      defineReactive(obj, keys[i])
    }
  }
  observeArray (items: Array<any>) {
    for (let i = 0, l = items.length; i < l; i++) {
      observe(items[i])
    }
  }
}

通过递归的方式对data中的每一项生成get和set。如果data中有数组，会调用observeArray方法，循环每一项，再调用observe(每一项的值)

在浏览器中测试查看：
通过Vue.observable传入一个对象，如果这个对象中嵌套了对象，会递归生成getter、setter。

在index.js中还有set和del两个函数，这两个函数可以用在vue的实例$set和$delete上。set函数可以添加一个属性到一个对象中。如果这个属性不存在的话，会触发notify更新。

// 在vue初始化中，会将 set赋值为实例方法$set
Vue.prototype.$set = set
Vue.prototype.$delete = del

总结： 通过observe方法为对象中的属性通过Object.definedProperty添加get和set。

2.2 Dep

用于依赖的收集，和更新。
参考： https://www.bilibili.com/video/BV1d4411v7UX?p=4

简单的实现：

let activeUpdate = null;
// 创建跟踪依赖
class Dep {
   constructor() {
      this.subscribers = new Set();
   }
   depend() {
      if (activeUpdate) {
         this.subscribers.add(activeUpdate)
      }
   }
   notify() {
      this.subscribers.forEach((sub) => { sub() })
   }
}

// autorun 函数，接收一个更新函数
function autorun(func) {
   function wrappedFunc() {   // 会不断的更新依赖
      activeUpdate = func;
      func()
      activeUpdate = null;
   }
   wrappedFunc();
}

// 实现效果
const dep = new Dep();
autorun(() => {
   dep.depend();  // 收集依赖
   console.log("update！");
})

dep.notify() // 触发autorun

在通过observe为对象添加了get和set之后，通过在get函数中收集依赖dep.depend()在set中触发更新dep.notify()。如在index.js中的defineReactive函数中：

/**
   * 通过get 添加依赖 dep.depend() 
   * 通过set 执行更新 dep.notify()
   */
Object.defineProperty(obj, key, {
   enumerable: true,
   configurable: true,
   get: function reactiveGetter () {
   const value = getter ? getter.call(obj) : val
   if (Dep.target) {
      dep.depend()  // 添加依赖，通过dep.notify()更新
      if (childOb) {
         childOb.dep.depend()
         if (Array.isArray(value)) {
         dependArray(value)
         }
      }
   }
   return value
   },
   set: function reactiveSetter (newVal) {
      const value = getter ? getter.call(obj) : val
      /* eslint-disable no-self-compare */
      if (newVal === value || (newVal !== newVal && value !== value)) {
         return
      }
      /* eslint-enable no-self-compare */
      if (process.env.NODE_ENV !== 'production' && customSetter) {
         customSetter()
      }
      // #7981: for accessor properties without setter
      if (getter && !setter) return
      if (setter) {
         setter.call(obj, newVal)
      } else {
         val = newVal
      }
      childOb = !shallow && observe(newVal)
      // 【在set中触发更新】
      dep.notify()
   }
})

2.2 Watcher

通过Watcher记录是谁调用的get方法。

在dep中，有一个Dep.target表示当前的watcher的全局变量，在dep.depend()收集依赖时，会查看这个全局变量，以确定当前的依赖。会把当前的依赖添加到一个watcher中，如果通过set函数执行了dep.notify()会将watcher添加到scheduler的一个队列中去。这个队列主要用于去除同步任务中重复添加的watcher，通过id去除。最后这个watcher列表中的每一个watcher执行一个run方法。

export default class Dep {
  static target: ?Watcher;
  id: number;
  subs: Array<Watcher>;

  constructor() {
    this.id = uid++
    this.subs = []
  }

  addSub(sub: Watcher) {
    this.subs.push(sub)
  }

  removeSub(sub: Watcher) {
    remove(this.subs, sub)
  }

  depend() {
    if (Dep.target) {
      // 添加挡墙的Dep.target
      Dep.target.addDep(this)
    }
  }

  notify() {
    // stabilize the subscriber list first
    const subs = this.subs.slice()
    if (process.env.NODE_ENV !== 'production' && !config.async) {
      // subs aren't sorted in scheduler if not running async
      // we need to sort them now to make sure they fire in correct
      // order
      subs.sort((a, b) => a.id - b.id)
    }
    for (let i = 0, l = subs.length; i < l; i++) {
      subs[i].update()
    }
  }
}
